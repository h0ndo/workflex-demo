# Project Structure Rules

## Required Structure

- `agents/`
- `rules/`
- `doku/`
- `requirements/`
- `prompt-history/`
- `e2e-tests/`
- `backend/`
- `frontend/`
- `.github/workflows/`

## Structural Conventions

- Use domain-first first-layer folders.
- Put technical subfolders inside each domain folder.
- Apply this convention consistently in backend, frontend libraries, and E2E tests.

## Language Policy

- Chat language can be German.
- Everything written into the repository must be English.
- Prompt-history entries must be written in English.
