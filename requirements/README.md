# Requirements

Dieser Ordner ist das zentrale Anforderungs-Archiv für Workflex.

## Ziel
Alle durch Requirements Engineering erarbeiteten Inhalte dauerhaft und nachvollziehbar dokumentieren.

## Struktur

- `LASTENHEFT.md` -> aktueller fachlicher Ist-Stand (bereits verfügbare Features)
- `PFLICHTENHEFT.md` -> geplanter Soll-Stand (geplante Features/Stories)
- `entries/` -> detaillierte Einzelanforderungen (pro Feature/Story)

## Prozess

1. Neue oder geänderte Anforderung wird in `entries/` dokumentiert.
2. Relevante Punkte werden in `PFLICHTENHEFT.md` aufgenommen oder aktualisiert.
3. Nach Umsetzung wandert der Status in Richtung `LASTENHEFT.md` (Ist-Stand).
4. Verweise zu GitHub Issues/Stories werden mitgeführt.
