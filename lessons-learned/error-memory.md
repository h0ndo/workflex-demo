# Error Memory

Track recurring or notable errors and the concrete action that resolved them.

Goal: reduce future error frequency by turning each incident into a reusable preventive pattern.

| Error (short) | Fix action |
| --- | --- |
| _Example: Quality pipeline failed on Angular ESLint `prefer-inject`_ | Refactored constructor injection to `inject()` in affected frontend services/components and re-ran lint checks. |
